# Product

## Register

brand

## Users

三类使用者：

- **读者（公开访客）**：通过链接直接到达某篇文档，独立阅读，不登录，不感知系统本身。阅读环境多样：桌面浏览器、偶尔移动端。
- **团队成员（4-5人）**：创作者和内容管理者，熟悉工具，期待系统是可信赖的工作环境，而非一个需要被适应的粗糙工具。
- **管理员**：维护空间、目录、权限、删除恢复和 skill 版本。

## Product Purpose

一个供小团队托管外部生成 HTML 文档的内部系统：把技术文档、团队文档、个人文档和项目资料按空间/目录组织起来，通过 iframe 安全隔离展示，并按权限对外分享。

成功标准：读者打开一篇文档，感受到的是内容本身，而不是"又一个 CMS"；团队成员打开管理后台，感受到的是工具对他们工作方式的尊重。

## Brand Personality

克制、编辑感、值得信任。

不是"极简主义"，而是有选择性的密度，每个元素都有理由在那里。不追求惊喜，追求让人觉得"理所应当如此"的品质。

## Anti-references

- **Notion / Confluence**：功能堆砌，块编辑器的自由度稀释了设计张力。
- **WordPress / Ghost 博客**：博客风格的版式，不够内敛，过于通用。
- **GitBook / ReadtheDocs**：技术文档的冷硬感，设计退到了功能后面。
- **企业内网 / SharePoint**：无设计语言，组织感压过了内容感。

## Design Principles

1. **内容是唯一主角**：版式、色彩、动效全部服务于文字的可读性，不争夺注意力。
2. **克制即品质**：任何元素如果去掉后页面更好，就去掉。装饰要有理由。
3. **系统感而非工具感**：设计决策全局一致，读者和编辑都在同一套语言里工作。
4. **暖而不腻**：用暖色调营造阅读舒适感，但不滑向"温馨"或"可爱"，保持专业距离。
5. **交互克制，反馈清晰**：动效轻量且有目的，操作结果立刻可见，不靠动效填充空洞。

## Accessibility & Inclusion

- WCAG AA 合规（对比度、焦点态）。
- 关键交互键盘可操作，Tab 顺序合理。
- 图标按钮有 `aria-label`，表单元素有关联 `label`。
- 尊重 `prefers-reduced-motion`：动效在系统设置减少动效时降级或关闭。
