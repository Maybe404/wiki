# V2.0 整改执行计划

> 本轮聚焦交付，**不是** 完整的 V2 蓝图。配套两份资产：
> - `skills/document-generation-v0.1.md` — Skill v0.1 起稿（独立交付物）
> - `CLAUDE.md` / `PRODUCT.md` / `DESIGN.md` — 沿用，本计划与之冲突时以本计划为准

## 0. 本轮目标与边界

### 0.1 做什么

把现有系统改造成：「外部生成 HTML（由人 + Skill + Claude Code/Codex/Claude.ai）→ 粘贴/上传到系统 → iframe 沙箱渲染 → 按空间/目录/权限组织管理」这个最小闭环。

- 阅读页改为 **iframe sandbox** 渲染，对粘贴或导入的 HTML 不做风格校验，最大兼容
- 新增 **空间 / 目录 / 文档** 三层结构
- 新增 **游客 / 成员 / 管理员** 三类权限
- 新增 **Skill 资产管理**（上传、激活、对外公开下载）
- 提供 **Skill v0.1 起稿**（系统首次启动时自动 seed）

### 0.2 不做什么（本轮明确推迟）

- 系统内置 AI 生成（暂不接 Anthropic SDK，外部生成已够用）
- AI 对话式精修
- 内联文字编辑、所有富文本编辑
- 版本管理与版本对比（每篇文档暂存最新一份 HTML，re-paste 会覆盖；但保留最近 5 份快照防误改，无 UI）
- 全文搜索增强（仅按标题搜）
- 兼容性硬校验报告（最多只做"这是不是 HTML"的语法检查）
- 视觉 diff / 截图服务

## 2. 数据模型变更

按以下结构落地。除非有不可调和的冲突，遵循这个设计。

### 2.1 User

扩展现有 User 模型：

```python
class User(AbstractUser):
    role = CharField(max_length=20,
                     choices=[("admin", "Admin"), ("member", "Member")],
                     default="member")
    # 注：游客无账号，仅通过公开链接访问
```

### 2.2 Workspace

```python
class Workspace(models.Model):
    id = UUIDField(primary_key=True, default=uuid.uuid4)
    name = CharField(max_length=100)
    slug = SlugField(max_length=60, unique=True)
    description = TextField(blank=True)
    created_by = ForeignKey(User, on_delete=PROTECT, related_name="created_workspaces")
    created_at = DateTimeField(auto_now_add=True)
    is_deleted = BooleanField(default=False)
```

### 2.3 WorkspaceMember

```python
class WorkspaceMember(models.Model):
    workspace = ForeignKey(Workspace, on_delete=CASCADE, related_name="members")
    user = ForeignKey(User, on_delete=CASCADE, related_name="workspace_memberships")
    can_edit = BooleanField(default=True)  # False = 只读成员
    joined_at = DateTimeField(auto_now_add=True)
    
    class Meta:
        unique_together = [("workspace", "user")]
```

`admin` 角色的 User 隐式拥有所有 Workspace 的访问权，**不需要**写 WorkspaceMember 记录。

### 2.4 Folder

```python
class Folder(models.Model):
    id = UUIDField(primary_key=True, default=uuid.uuid4)
    workspace = ForeignKey(Workspace, on_delete=CASCADE, related_name="folders")
    parent = ForeignKey("self", null=True, blank=True, 
                        on_delete=CASCADE, related_name="children")
    name = CharField(max_length=100)
    slug = SlugField(max_length=60)
    created_by = ForeignKey(User, on_delete=PROTECT)
    created_at = DateTimeField(auto_now_add=True)
    
    class Meta:
        unique_together = [("workspace", "parent", "slug")]
```

允许嵌套，深度建议 ≤ 3 层（前端不强制，但建议）。

### 2.5 Document

```python
class Document(models.Model):
    id = UUIDField(primary_key=True, default=uuid.uuid4)
    workspace = ForeignKey(Workspace, on_delete=PROTECT, related_name="documents")
    folder = ForeignKey(Folder, null=True, blank=True, 
                        on_delete=SET_NULL, related_name="documents")
    
    title = CharField(max_length=200)
    slug = SlugField(max_length=80)
    html = TextField()                  # 完整 HTML 文档，原样存储
    plain_text = TextField(blank=True)  # 从 html 抽取，仅供搜索
    
    visibility = CharField(max_length=20, choices=[
        ("private", "Private"),         # 仅 owner + admin
        ("workspace", "Workspace"),     # workspace 内成员可见
        ("link_shared", "Link shared"), # 任何人通过链接可读
    ], default="workspace")
    
    owner = ForeignKey(User, on_delete=PROTECT, related_name="documents")
    
    created_at = DateTimeField(auto_now_add=True)
    updated_at = DateTimeField(auto_now=True)
    is_deleted = BooleanField(default=False)
    deleted_at = DateTimeField(null=True, blank=True)
    
    class Meta:
        unique_together = [("workspace", "slug")]
```

### 2.6 DocumentSnapshot（轻量历史，无 UI）

```python
class DocumentSnapshot(models.Model):
    document = ForeignKey(Document, on_delete=CASCADE, related_name="snapshots")
    html = TextField()
    created_at = DateTimeField(auto_now_add=True)
    created_by = ForeignKey(User, on_delete=PROTECT)
    
    class Meta:
        ordering = ["-created_at"]
```

每次 Document.html 被覆盖前自动生成一条 snapshot。Manager 默认只保留最近 5 条，超过自动清理。**本期不提供 UI 查看，纯防误删兜底**。

### 2.7 SkillVersion

```python
class SkillVersion(models.Model):
    version = CharField(max_length=20, unique=True)   # v0.1, v0.2, ...
    content = TextField()                              # markdown
    notes = TextField(blank=True)
    is_active = BooleanField(default=False)
    created_by = ForeignKey(User, on_delete=PROTECT)
    created_at = DateTimeField(auto_now_add=True)
    
    class Meta:
        constraints = [
            UniqueConstraint(fields=["is_active"],
                             condition=Q(is_active=True),
                             name="single_active_skill"),
        ]
```

## 5. 改造清单

### 5.1 阅读页改 iframe sandbox（核心）

路由分两层：

| 路由 | 内容 |
|---|---|
| `/d/<workspace-slug>/<doc-slug>` | 外壳页（系统横条 + iframe 引用） |
| `/d/<workspace-slug>/<doc-slug>/content` | 文档完整 HTML，配 CSP 头 |

外壳页关键代码：

```html
<iframe
    src="{{ content_url }}"
    sandbox="allow-scripts"
    referrerpolicy="no-referrer"
    style="width:100%; border:none; display:block;"
    title="{{ doc.title }}">
</iframe>
<script>
window.addEventListener("message", (e) => {
    if (e.data?.type === "doc-resize") {
        document.querySelector("iframe").style.height = e.data.height + "px";
    }
});
</script>
```

`/content` 视图配的 CSP 头：

```python
response["Content-Security-Policy"] = (
    "default-src 'none'; "
    "script-src 'unsafe-inline' 'unsafe-eval' "
        "https://cdn.jsdelivr.net https://unpkg.com https://cdnjs.cloudflare.com; "
    "style-src 'unsafe-inline' https://fonts.googleapis.com; "
    "font-src https://fonts.gstatic.com data:; "
    "img-src data: https:; "
    "connect-src 'none';"
)
response["X-Frame-Options"] = "SAMEORIGIN"
```

**关键安全点**：iframe `sandbox="allow-scripts"` 但**绝对不含** `allow-same-origin`。这是整个安全模型的底线。

返回的 HTML 在 `</body>` 前注入一段固定的高度通信脚本（约 10 行 vanilla JS）让 iframe 通过 `postMessage` 告诉父页面自己的高度。

### 5.2 文档创建：粘贴或上传

简化形态：

- 标题（必填）
- 选择 Workspace（默认当前）
- 选择 Folder（可选）
- HTML 内容：粘贴文本框 / 上传 `.html` 文件二选一
- 可见性：private / workspace / link_shared

提交后：
- 仅做最基础检查：能不能被 BeautifulSoup 解析、长度在 1KB–2MB 之间
- 不强制 skill 签名、不强制 token 引用、不强制响应式
- 抽取 `<body>` 内的纯文本存到 `plain_text` 用于搜索
- 直接保存为 Document.html，slug 自动生成（标题 slugify，冲突加序号）

**不要做兼容性报告**。用户说了：「不要规定死规则，因为后面我可能会迁入一些风格不同的页面文档进来」。

### 5.3 列表与导航

- 主页 `/` → 当前用户能访问的 Workspace 列表
- `/w/<workspace-slug>/` → 该空间的 Folder + Document 列表（树状或两栏）
- `/w/<workspace-slug>/f/<folder-slug>/` → 该目录下的内容
- 顶部面包屑：Workspace › Folder › Document
- 搜索框：当前 workspace 范围内按标题搜（V2.1 再扩 plain_text）

### 5.4 Skill 管理后台（admin only）

`/admin/skills/`：

- 当前激活 skill：版本号、激活时间、查看、下载
- 历史版本表：版本、上传时间、备注、激活按钮、查看、下载
- 上传新 skill：选 `.md` 文件 → 填版本号（建议 v0.2 这种）→ 提交

公开下载端点（**无需登录**）：

- `GET /skill/current.md` → 当前激活 skill
- `GET /skill/v<n>.md` → 指定版本

响应头加 `Content-Disposition: attachment; filename="skill-v0.1.md"`。

## 6. 新增清单

### 6.1 Workspace CRUD（admin only）

- 创建：填名称、描述、slug
- 编辑：名称、描述、添加/移除成员、调整成员 can_edit
- 删除：软删除（is_deleted=True）

### 6.2 Folder CRUD（workspace 内可编辑成员可操作）

- 创建：在当前位置创建子目录
- 重命名
- 删除：仅当目录为空时允许（保护文档）
- 移动：拖拽或选目标目录（不强求拖拽 UI，下拉选择也行）

### 6.3 文档管理操作

- **复制**：在当前 workspace 内复制一份 Document（slug 加 -copy 后缀），HTML 完整复制
- **移动**：跨 Workspace / Folder 移动（owner 操作自己的文档，admin 操作所有）
- **软删除**：移入回收站，30 天自动清理
- **永久删除**：admin 在回收站可立即清除

### 6.4 权限矩阵

| 操作 | 游客 | 成员 | 管理员 |
|---|---|---|---|
| 阅读 link_shared 文档（直链） | ✓ | ✓ | ✓ |
| 阅读 workspace 文档 | ✗ | 仅成员所在 workspace | ✓ |
| 阅读 private 文档 | ✗ | 仅 owner | ✓ |
| 创建文档 | ✗ | ✓（有 can_edit 的 workspace 内） | ✓ |
| 编辑文档 | ✗ | owner 或 workspace 内 can_edit | ✓ |
| 复制 / 移动文档 | ✗ | owner | ✓ |
| 软删除文档 | ✗ | owner | ✓ |
| 永久删除文档 | ✗ | ✗ | ✓ |
| 创建 Workspace | ✗ | ✗ | ✓ |
| 管理 Workspace 成员 | ✗ | ✗ | ✓ |
| Skill 管理 | ✗ | ✗ | ✓ |
| 下载 skill | ✓ | ✓ | ✓ |

### 6.5 Skill v0.1 自动 seed

系统首次启动 / migration 后自动加载 `skills/document-generation-v0.1.md` 的内容到 SkillVersion 并激活。

**Claude Code 不要尝试编写或修改 skill 内容**。它是产品方手工撰写的资产，本计划同目录下交付了起稿文件，直接 seed 即可。

## 8. 验收标准

本轮全部完成后必须满足：

1. 在 admin 账户下，能创建 Workspace、邀请 member、在 Workspace 内建 Folder、创建 Document
2. 把 `/mnt/example` 下任意一份纯 HTML（含 `<style>` 和 `<script>`）粘贴进系统，`/d/<workspace>/<doc>` 能完美展示
3. 在 iframe 内的 `<script>` 中执行 `document.cookie` 与 `fetch('/api/some-endpoint')`，两者都被沙箱+CSP 拦截，父页面不受影响
4. 用 member 账户登录，能看到自己 workspace 的内容，看不到其他 workspace
5. 用未登录浏览器访问 `link_shared` 文档的链接能正常阅读
6. 未登录访问 `/skill/current.md` 能下载到当前激活的 skill markdown 文件
7. 软删除文档后从列表消失；恢复后回到原位置
8. 复制文档后产生新 slug，HTML 内容完全相同，独立可编辑
9. CLAUDE.md 中过时条目已同步修订（详见 §10）

## 9. 不做清单（防需求蔓延）

- 系统内置 AI 生成（暂不接 Anthropic SDK）
- 完整的版本历史 UI（只有兜底 5 份快照，无 UI）
