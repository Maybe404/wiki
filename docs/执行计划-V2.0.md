# V2.0 整改执行计划

> 本轮目标是把现有 Atlas 改成「外部生成完整 HTML → 粘贴/上传 → iframe 隔离展示 → 按空间/目录/权限管理」的最小闭环。
>
> 重要原则：**不重建项目，不推翻现有首页和后台审美，不把文档生成规则锁死。** 现有系统已经有 Django app、treebeard 文档树、HTML 导入、完整 HTML iframe 雏形和 Claude 暖白视觉系统，本计划基于这些能力增量改造。

## 0. 目标与边界

### 0.1 本轮必须交付

- 外部完整 HTML 可以粘贴或上传，保留 `<style>`、`<script>`、字体、图形和动效能力。
- 阅读页使用 iframe sandbox 隔离渲染，保护父页面、登录态、后台接口和系统样式。
- 保留现有 Atlas 首页、暖白底色、珊瑚行动色、文档式后台，不改成泛 SaaS 管理台。
- 增加 Workspace 概念，并把现有目录树限定到 workspace 内。
- 增加游客、成员、管理员三类权限。
- 增加文档复制、移动、删除管理能力。其中删除默认仅管理员可执行。
- 增加 Skill 版本管理和公开下载端点。
- 保留未来接入不同风格页面的余地，不做风格硬校验。

### 0.2 本轮明确不做

- 不做系统内置 AI 生成，不接 Anthropic/OpenAI SDK。
- 不做内联文字编辑、富文本编辑、HTML 在线编辑。
- 不做版本 diff UI、视觉 diff、截图服务。
- 不做复杂兼容性报告。
- 不做 SSO、组织树、评论、协作提醒。
- 不把所有外部 HTML 强制改成 Atlas 视觉。
- 不改成自定义 User 模型，避免 Django 迁移风险。

## 1. 项目现状锚点

开工前执行者必须先读代码并输出盘点报告，确认以下事实是否仍成立：

- `apps.documents.models.Document` 当前继承 `treebeard.mp_tree.MP_Node`，并用 `node_type=document/folder` 承载文档树。
- `DocumentVersion.html` 当前保存 HTML 内容，`is_full_page=True` 已表示完整 HTML 页面。
- 公开页和后台详情页已经存在 iframe `srcdoc` 渲染完整 HTML 的雏形。
- `apps.editor.validator.sanitize_full_page()` 当前会移除 `<script>`，这与 V2 “原样展示动效页面”目标冲突。
- 当前项目使用默认 Django User，未配置 `AUTH_USER_MODEL`。
- 根目录 `DESIGN.md` 是设计系统权威文件，不是 `docs/DESIGN.md`。
- `static/css/public.css` 和 `static/css/admin.css` 已落地 Claude 暖白 + 珊瑚色 token。

盘点报告输出「保留 / 删除 / 改造 / 新增」四个清单，每条带文件路径。**报告交付后停下，等用户确认再动代码。**

## 2. 总体技术策略

### 2.1 保留现有 Document 树

不新增独立 `Folder` 模型。继续使用现有 `Document(MP_Node)`：

- `node_type="folder"` 表示目录。
- `node_type="document"` 表示文档页。
- 移动目录和文档继续使用 treebeard `move()`。
- workspace 作为 `Document` 上的新字段，所有树查询都按 workspace 过滤。

这样可以复用现有后台目录树、拖拽移动、模板标签和 treebeard 迁移基础，避免把项目拆掉重建。

### 2.2 保留默认 Django User

不创建 `class User(AbstractUser)`。角色规则：

- 管理员：`is_superuser=True` 或 `is_staff=True`。
- 成员：存在 `WorkspaceMember` 记录的登录用户。
- 游客：未登录用户，仅能访问 `link_shared` 文档和公开 skill 下载。

如后续需要更细的全局角色，再加 `UserProfile`，不要在 V2 中切换 `AUTH_USER_MODEL`。

### 2.3 内容存储沿用 DocumentVersion，先不搬家

不在本轮强行把 HTML 从 `DocumentVersion.html` 搬到 `Document.html`。

本轮语义调整为：

- 当前文档内容 = 最新一条非 auto 的 `DocumentVersion`，没有则取最新版本。
- 新导入/重新粘贴完整 HTML 时，创建一条新的 `DocumentVersion`。
- 只保留最近 5 条内容快照，无版本 diff UI。
- `editable_blocks` 字段保留兼容旧数据，但新完整 HTML 文档不再依赖它。

后续如果确认版本能力彻底不要，再单独做一次数据瘦身。

## 3. 数据模型变更

### 3.1 Workspace

```python
class Workspace(models.Model):
    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
    name = models.CharField(max_length=100)
    slug = models.SlugField(max_length=60, unique=True, allow_unicode=True)
    description = models.TextField(blank=True)
    created_by = models.ForeignKey(User, on_delete=models.PROTECT, related_name="created_workspaces")
    is_deleted = models.BooleanField(default=False)
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
```

### 3.2 WorkspaceMember

```python
class WorkspaceMember(models.Model):
    workspace = models.ForeignKey(Workspace, on_delete=models.CASCADE, related_name="members")
    user = models.ForeignKey(User, on_delete=models.CASCADE, related_name="workspace_memberships")
    can_edit = models.BooleanField(default=True)
    joined_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        constraints = [
            models.UniqueConstraint(fields=["workspace", "user"], name="unique_workspace_member")
        ]
```

管理员隐式拥有全部 workspace 权限，不需要写 membership 记录。

### 3.3 Document 增量字段

在现有 `Document` 上新增：

```python
workspace = models.ForeignKey(
    Workspace,
    on_delete=models.PROTECT,
    related_name="nodes",
    null=True,  # 迁移期允许为空，数据迁移后改为 null=False
)
visibility = models.CharField(
    max_length=20,
    choices=[
        ("private", "Private"),
        ("workspace", "Workspace"),
        ("link_shared", "Link shared"),
    ],
    default="workspace",
)
plain_text = models.TextField(blank=True)
deleted_at = models.DateTimeField(null=True, blank=True)
```

slug 规则：

- V2 路由使用 `/d/<workspace-slug>/<doc-slug>/`，因此 slug 不再需要全站唯一。
- 首期为了降低迁移风险，可以先保持现有全局唯一 slug。
- 第二步再把唯一约束改成 `(workspace, slug)`，并同步调整 `SlugAlias` 增加 workspace 字段。

### 3.4 SkillVersion

```python
class SkillVersion(models.Model):
    version = models.CharField(max_length=20, unique=True)
    content = models.TextField()
    notes = models.TextField(blank=True)
    is_active = models.BooleanField(default=False)
    created_by = models.ForeignKey(User, on_delete=models.PROTECT)
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        constraints = [
            models.UniqueConstraint(
                fields=["is_active"],
                condition=models.Q(is_active=True),
                name="single_active_skill",
            )
        ]
```

seed 文件路径必须使用实际路径：`skills/html-document/SKILL.md`。

## 4. iframe 渲染改造

### 4.1 路由

| 路由 | 用途 |
|---|---|
| `/d/<workspace-slug>/<doc-slug>/` | Atlas 外壳页：顶部/面包屑/权限判断/iframe |
| `/d/<workspace-slug>/<doc-slug>/content/` | 返回文档完整 HTML，并设置文档专用 CSP |
| `/d/<slug>/` | 旧链接兼容：能定位则 301 到新路由 |

### 4.2 外壳页

外壳页属于 Atlas 系统，继续使用现有暖白视觉。iframe 示例：

```html
<iframe
  id="doc-frame"
  class="pub-doc-frame"
  src="{{ content_url }}"
  sandbox="allow-scripts allow-downloads allow-popups"
  referrerpolicy="no-referrer"
  title="{{ document.title }}">
</iframe>
```

底线：

- **绝对不加 `allow-same-origin`**。
- 不使用 `srcdoc` 作为主方案，改用 `/content/`，方便设置 CSP、缓存和下载策略。
- 父页面监听 `postMessage` 时必须校验 `event.source === iframe.contentWindow`。
- iframe 高度更新必须限制范围，例如 `200 <= height <= 50000`。

### 4.3 content 视图

`/content/` 返回最新 `DocumentVersion.html`，并在返回前注入 resize 脚本：

```js
(() => {
  const send = () => parent.postMessage({
    type: "atlas-doc-resize",
    height: Math.ceil(document.documentElement.scrollHeight)
  }, "*");
  new ResizeObserver(send).observe(document.documentElement);
  window.addEventListener("load", send);
  send();
})();
```

注入策略：

- 有 `</body>` 时插入到 `</body>` 前。
- 没有 `</body>` 时追加到末尾。
- 不解析重写用户 HTML 结构，不做美学修正。

### 4.4 CSP 策略

安全模型的重点是保护 Atlas 父系统，而不是把文档页面变丑。

默认使用 `standard` 策略，适合 AI 生成的美观技术文档：

```python
Content-Security-Policy:
  default-src 'none';
  script-src 'unsafe-inline' 'unsafe-eval'
    https://cdn.jsdelivr.net https://unpkg.com https://cdnjs.cloudflare.com;
  style-src 'unsafe-inline'
    https://fonts.googleapis.com https://cdn.jsdelivr.net https://unpkg.com https://cdnjs.cloudflare.com;
  font-src https://fonts.gstatic.com https://cdn.jsdelivr.net https://unpkg.com data:;
  img-src https: data: blob:;
  media-src https: data: blob:;
  connect-src https:;
  frame-src https:;
```

可选增加 `resource_policy` 字段：

- `standard`：默认，允许常见 CDN、字体、图片、有限外部请求。
- `strict`：用于不可信 HTML，关闭外部脚本和 connect。
- `permissive`：用于迁入第三方项目文档，允许更多 `https:` 资源。

V2 首期可以只实现 `standard`，但代码结构要便于后续加策略。

### 4.5 X-Frame-Options

全局 `X_FRAME_OPTIONS = "DENY"` 保留。`/content/` 响应单独设置：

```python
response["X-Frame-Options"] = "SAMEORIGIN"
```

需要手测确认 Django `XFrameOptionsMiddleware` 没有覆盖该 header。

## 5. HTML 导入改造

当前导入逻辑会提取正文、移除脚本和样式、添加 `data-editable`，这与 V2 目标冲突。

新导入页表单：

- 标题。
- Workspace。
- 目录，可为空。
- 可见性：`private / workspace / link_shared`。
- HTML 内容：粘贴文本框或上传 `.html/.htm`。
- 资源策略：先隐藏，默认 `standard`；后续需要再暴露。

提交后只做基础检查：

- 文件大小建议 1KB 到 2MB，超出给明确错误。
- 能被 BeautifulSoup 解析。
- 至少包含一段可见文本，或用户确认“这是纯可视化页面”。
- 抽取纯文本写入 `plain_text`。
- 直接保存完整 HTML 到 `DocumentVersion.html`，`is_full_page=True`。

不做：

- 不强制 skill 签名。
- 不强制 token。
- 不强制响应式。
- 不删除 `<style>` / `<script>`。
- 不注入 `data-editable`。
- 不把外部页面改造成 Atlas 样式。

## 6. Workspace、目录与导航

### 6.1 首页

保留现有 `/` 首页风格，不改成工作区列表。

登录后：

- `/admin/` 显示当前用户可访问的 workspace 列表或默认进入最近 workspace。
- 顶部或侧栏提供 workspace switcher。

公开访问：

- `link_shared` 文档可直接访问 `/d/<workspace>/<doc>/`。
- 未登录用户不能看到 workspace 列表。

### 6.2 Workspace 页

新增：

| 路由 | 用途 |
|---|---|
| `/w/<workspace-slug>/` | workspace 内目录和文档列表 |
| `/w/<workspace-slug>/trash/` | 回收站，管理员可见 |

目录仍由 `Document.node_type="folder"` 表示。后台树查询必须加：

```python
Document.get_tree().filter(workspace=current_workspace, is_deleted=False)
```

### 6.3 面包屑

文档外壳页显示：

```text
Workspace › Folder › Document
```

如果 iframe 页面本身已经有强视觉标题，Atlas 外壳的标题区要克制，避免抢文档内容的第一视觉。

## 8. 文档管理操作

### 8.1 复制

- 复制最新完整 HTML。
- 新文档默认与源文档同 workspace、同目录。
- slug 追加 `-copy`，冲突追加序号。
- owner = 当前操作者。
- visibility 默认继承源文档，也可以在复制弹窗里修改。

### 8.2 移动

- 支持同 workspace 内移动目录位置。
- 支持跨 workspace 移动文档；移动目录跨 workspace 放到 V2.1，避免 treebeard 子树迁移风险过大。
- 移动时校验目标 workspace 权限、slug 冲突和目录类型。

### 8.3 删除

- 管理员软删除。
- 删除文档：设置 `is_deleted=True`、`deleted_at=now()`。
- 删除目录：首期只允许空目录；如要删除非空目录，必须二次确认并软删除整棵子树。
- 回收站中可恢复或永久删除。
- 自动清理 30 天前软删除内容可以先做 management command，不必首期接定时任务。

## 9. Skill 管理

新增 `/admin/skills/`，管理员可用：

- 查看当前激活 skill。
- 上传 `.md` 新版本。
- 填写版本号和备注。
- 激活某个历史版本。
- 下载任意版本。

公开端点：

- `GET /skill/current.md`
- `GET /skill/<version>.md`

响应头：

```python
Content-Disposition: attachment; filename="skill-v0.1.md"
Content-Type: text/markdown; charset=utf-8
```

首次 migration 或管理命令 seed：

- 读取 `skills/html-document/SKILL.md`。
- 创建 `SkillVersion(version="v0.1", is_active=True)`。
- 如果没有管理员用户，seed 阶段应失败并提示先创建 superuser，不要静默创建虚假用户。

## 10. 删除与清理策略

不要一开始就大删文件。按功能退场：

第一阶段保留但隐藏：

- 内联编辑按钮。
- version diff 抽屉入口。
- `data-editable` 导入提示。

第二阶段确认新流程稳定后删除：

- `static/js/editor.js`
- `static/js/versions.js`
- 编辑保存 API。
- diff API。
- `data-editable` 自动注入逻辑。

`bleach` 保留：

- 标题、备注、用户名、skill notes 等人类填写字段可继续清洗或转义。
- 文档主体 HTML 不走 bleach，靠 iframe sandbox + CSP 隔离。

## 11. 推进节奏

建议 7 个阶段，每阶段独立验证。

| 阶段 | 内容 | 验证点 |
|---|---|---|
| 0 | 现状盘点报告 | 用户确认后再动代码 |
| 1 | iframe `/content/` + 原样 HTML 导入 | 带 CSS/JS 的 HTML 能展示、动效能运行、父页面 cookie 不可读 |
| 2 | Workspace + membership 模型迁移 | 现有文档进入“默认空间”，后台能按 workspace 过滤 |
| 3 | 权限判断与新路由 | 游客/成员/管理员访问矩阵符合预期 |
| 4 | 管理端 workspace switcher、目录树、面包屑 | 首页风格不变，后台保持现有暖白文档工具感 |
| 5 | 复制、移动、删除、回收站 | slug 冲突、权限、恢复流程可手测 |
| 6 | SkillVersion 管理与公开下载 | `/skill/current.md` 未登录可下载 |
| 7 | 隐藏/清理旧编辑与 diff 功能 | 老入口不可见，旧数据仍可读 |

每阶段前后：

```bash
mkdir -p backups
cp data/db.sqlite3 backups/pre-v2-$(date +%Y%m%d-%H%M%S).db
uv run ruff format .
uv run ruff check . --fix
uv run ty check
uv run pytest
```

Mac 本地手测文件建议放在：

```text
/private/tmp/atlas-html-fixtures/
```

不要使用 Linux 风格 `/mnt/example` 作为验收路径。

## 12. 验收标准

V2 完成后必须满足：

1. 现有首页 `/` 仍保持 Atlas 暖白 + 珊瑚色视觉，不被 workspace 列表替代。
2. admin 能创建 workspace、添加成员、调整 can_edit。
3. admin/member 能在 workspace 内创建目录、导入完整 HTML 文档。
4. 粘贴含 `<style>`、`<script>`、Google Fonts、GSAP/Alpine 或内联 SVG 动效的 HTML 后，阅读页能完整展示。
5. iframe 内脚本不能读取父页面 DOM、cookie、localStorage。
6. iframe 内脚本不能调用同源后台接口并携带登录态。
7. member 只能看到自己 workspace 的内容。
8. 未登录用户只能访问 `link_shared` 文档。
9. 管理员可以复制、移动、删除、恢复文档。
10. `/skill/current.md` 未登录可下载当前激活 skill。
11. `CLAUDE.md` 中 DESIGN 路径、安全规则和 V2 不做项已经同步修订。

| 现有条目 | V2 修订 |
