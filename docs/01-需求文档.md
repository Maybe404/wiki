# 内部文档系统 · 需求文档

> 一个具有编辑感的、可托管 AI 生成内容的、可控的小型文档系统。
> 不是 Notion，不是 WordPress，不是 GitBook。

---

## 一、项目定位

### 1.1 一句话定义

一个供个人与 4–5 人小团队使用的内部文档系统，承载由 AI（Codex / Claude Code）生成的、视觉风格统一的 HTML 文档页面，支持在线文字编辑、版本管理与按需对外发布。

### 1.2 它要解决什么

- AI 生成的文档页面各自为政、风格漂移 → **由系统强制套用统一框架与设计 token**。
- 文档内容更新需要改 HTML 源码 → **页面上点击即编辑、改完即存，不碰源码**。
- 团队协作无法回溯改动 → **每次保存都是一个版本，可对比、可回退**。
- 内部/对外混在一起 → **草稿默认私密，主动发布后才公开可访问**。

### 1.3 不做的事

- 不做多人实时协同（4–5 人异步编辑足够）。
- 不做 Notion 式自由块编辑（结构由 HTML 决定，只改文字）。
- 不做评论、@、订阅、通知（保持轻）。
- 不做企业级 SSO、审批流、组织架构。
- 不做移动端编辑（移动端只读）。

## 二、角色与权限

### 2.1 三类角色

**管理员（Admin）**
- 唯一最高权限账号（项目所有者）。
- 拥有全部内容的增/删/改/发布/回退/账号管理权限。
- 可指派其他用户为编辑者。

**编辑者（Editor）**
- 团队成员账号（4–5 人）。
- 可登录查看全部内容（含草稿），可编辑分配给自己的文档。
- 不可删除、不可发布、不可管理账号。

**访客（Public）**
- 无需登录。
- 仅能访问已发布的公开链接。
- 看不到草稿、看不到编辑入口、看不到版本历史。

### 2.2 访问入口

| 路径 | 谁能看 | 说明 |
|---|---|---|
| `/d/<slug>` | 所有人（已发布的） | 公开文档阅读页 |
| `/login` | 所有人 | 不在主导航暴露，需知道地址 |
| `/admin/` | 管理员 / 编辑者 | 登录后的管理后台 |
| `/admin/edit/<id>` | 管理员 / 编辑者 | 编辑界面 |

公开页面**完全不出现"登录"链接**——保持对外的纯净感。管理者通过书签或自己记住 `/login` 进入。

## 三、核心功能

### 3.1 文档浏览（公开端）

每篇已发布文档对应一个独立 URL（`/d/<slug>`）。打开后是一个纯粹的阅读体验：

- 无顶部进度条。
- 主标题用衬线字体，居中或左对齐由文档自身决定。
- 内容主区域 760px 宽度，桌面端左右各 48px padding，平板 32px，移动端 20px。
- 段落、引用、卡片、对比块等元素严格遵循设计 token。
- 大型 section 可轻量 fade-up 进入（opacity 0→1，translateY 12px→0），不夸张。
- 页脚极简：仅显示最后更新日期，无 logo、无导航。

阅读页**没有任何登录入口、编辑入口**。读者就是读者。

### 3.2 目录与导航（管理端）

登录后进入 `/admin/`，左侧是固定侧边栏，右侧是当前选中文档的预览/编辑区。

**侧边栏**：
- 顶部：当前用户、搜索框（⌘K 唤起）。
- 中部：文档目录树，支持多级嵌套、拖拽排序。
- 文档项右侧 hover 显示三个动作图标（编辑 / 复制链接 / 更多）。
- 每篇文档前用小圆点标识状态：实心绿 = 已发布、空心 = 草稿、红 = 已归档。
- 底部：新建按钮（"+ 新文档"）+ 设置入口。

**目录树交互**：
- 点击文件夹展开/收起，状态在 session 内保持。
- 拖拽改变顺序与父子关系，松手立即保存（无"保存目录"按钮）。
- 右键（或长按）弹出菜单：重命名、新增子节点、移动、删除。

### 3.3 编辑体验（核心）

**进入编辑模式**：
- 文档预览页右上角"编辑"按钮，点击进入编辑模式。
- 进入后：所有 `data-editable` 区域出现 1px 虚线珊瑚色边框（primary 46% alpha），hover 时加 5% 珊瑚底色。
- 不可编辑区域（布局、装饰图、装饰编号）保持原样，鼠标悬停时显示锁标。
- 工具栏从顶部缓缓滑下，不遮挡内容。非编辑模式下编辑边界完全不出现。

**编辑操作**：
- 点击任意可编辑区域，光标即就位，可直接打字。
- 工具栏极简：**加粗、斜体、链接、清除格式、撤销、重做、保存**。
- 不提供改字体、改颜色、改大小的功能——风格由系统决定，不由编辑者决定。
- 按 `⌘S` / `Ctrl+S` 保存。
- 离开编辑模式前如有未保存改动，弹出确认提示。

**自动保存**：
- 停止打字 3 秒后自动保存为"自动版本"（不计入正式历史）。
- 手动保存创建"命名版本"，进入版本历史。
- 自动版本仅保留最近 5 个，超出自动清理。

**保存反馈**：
- 右下角浮出 toast：暗色底（`#181715`）浅色字，"已保存 · 14:32"。
- 2.5 秒后自动淡出。
- 失败时可用 danger-surface 底，仍保持 8px 圆角和简短文字，附"重试"按钮。

### 3.4 版本管理

每个文档维护一条完整的版本时间线：

**版本列表**：
- 编辑页右侧抽屉，点击"版本"图标滑出。
- 时间线倒序展示：每条版本包含时间、作者、变更摘要（增 X 字 / 删 Y 字）、可选的版本说明。
- 命名版本用衬线字体标注，自动版本用次级文字色淡化。

**版本对比**：
- 点击任一版本，主区域切换为"对比视图"。
- 左右双栏（移动端上下叠加）：左为当前版本，右为所选历史版本。
- 增加的文字用浅绿底色，删除的文字用浅红底色加删除线。
- 顶部一行操作：「← 返回 / 还原此版本 / 在新页面打开」。

**还原**：
- 点击"还原此版本"后，弹出二次确认（防误触）。
- 还原不是删除新版本，而是基于历史版本创建一个新版本，原有历史完整保留。

### 3.5 发布流程

**草稿与已发布**：
- 每篇文档有 `status` 字段：`draft` / `published` / `archived`。
- 默认 `draft`。
- 草稿仅登录后可见，公开 URL 返回 404。

**发布操作**：
- 编辑页顶部右侧的"发布"按钮。
- 点击后弹出抽屉确认：「即将发布到 `/d/<slug>`，所有人都将可访问」。
- 同步显示：当前 slug（可改）、首次发布的预计时间、是否生成新版本号。
- 确认后，状态切为 `published`，并自动创建一条命名版本（"发布于 YYYY-MM-DD"）。

**取消发布**：
- 已发布文档可"撤回"，状态回到 `draft`。
- 取消发布不删除文档，公开 URL 立即返回 404。

**链接管理**：
- slug 默认由文档标题生成（中文 → 拼音首字母 + 短随机后缀），可手动改。
- 改 slug 后旧链接自动 301 重定向到新链接（保留旧 slug 的映射记录）。

### 3.6 AI 生成内容的接入

这是整个系统**最特殊**的能力，区别于一切现成 CMS：

**接入方式**：
1. 用户在 Codex / Claude Code 中按预设模板生成 HTML 片段。
2. 在管理后台「+ 新文档」 → 选「从 HTML 导入」。
3. 粘贴 HTML 到大文本框，点"解析"。
4. 系统自动：
   - 清洗 HTML（剥离 `<script>`、危险属性，过 bleach）。
   - 校验是否符合系统 schema（必需 class、必需 `data-editable` 标记）。
   - 应用统一外层布局（840px 容器、字体、配色 token）。
   - 提取所有 `data-editable` 区域，存入版本表。
5. 校验通过后进入预览界面，可直接编辑、保存、发布。

**模板规范**（提供给 AI 的 prompt 模板，作为附件存在 `/templates/`）：
- 必须用系统提供的 class（`doc-section`, `doc-quote`, `doc-compare-pair` 等）。
- 文字内容必须包在 `data-editable="true"` 的元素里，且要有 `data-editable-id` 唯一标识。
- 不允许内联 `style` 属性。
- 不允许引入外部字体、外部 CSS。
- 不允许 `<script>`、`<iframe>`、`onclick` 等。

校验不通过时给出明确错误：哪一行、缺什么、怎么改。

### 3.7 搜索

- 全局搜索快捷键：`⌘K` / `Ctrl+K`。
- 唤起一个浮层（不是新页面），顶部居中，居中宽度 560px。
- 实时搜索（输入即查），结果分组：「文档标题」、「正文匹配」。
- 命中的关键词高亮。
- 上下箭头切换、回车跳转。
- 公开端不暴露搜索入口（保持阅读纯净）。

### 3.8 账号管理

**登录页**（`/login`）：
- 极简：居中卡片，账号 + 密码 + 登录按钮。
- 无注册入口（系统不开放注册）。
- 无"忘记密码"链接（管理员后台重置）。

**账号设置**（管理员）：
- `/admin/users` 路径下管理团队成员账号。
- 可新建、禁用、重置密码、删除。
- 每个账号可指定角色（admin / editor）。

**会话**：
- 登录默认 30 天 session（可勾选"本机记住"）。
- 修改密码后所有其他设备登录失效。

## 四、界面规范

> 界面规范以 `DESIGN.md` 为唯一权威来源，本节仅做功能层面的引用说明。实现时直接对照 `DESIGN.md` 中的 token。

### 4.1 配色 Token

| Token | 色值 | 用途 |
|---|---|---|
| `canvas` | `#faf9f5` | 全站底色，body、nav、阅读页 |
| `primary` | `#cc785c` | 唯一主行动色：按钮、链接、focus ring |
| `primary-hover` | `#b86950` | 主按钮 hover |
| `primary-pressed` | `#a9583e` | 主按钮 pressed，同时 scale(0.97) |
| `surface-soft` | `#f5f0e8` | callout、空状态、后台面板背景 |
| `surface-card` | `#efe9de` | 功能面板、tab active |
| `surface-dark` | `#181715` | 代码块、命令、AI 日志、重要 toast |
| `hairline` | `#e6dfd8` | 输入框、搜索框、顶栏下边界 |
| `hairline-soft` | `#ebe6df` | 表格行、列表分隔 |
| `ink` | `#141413` | 标题和关键 UI |
| `ink-body` | `#3d3d3a` | 正文默认色 |
| `ink-muted` | `#6c6a64` | meta、sidebar、helper text |
| `ink-subtle` | `#8e8b82` | placeholder、禁用态（不用于正文）|
| `ink-on-dark` | `#faf9f5` | 暗色代码块上的文字 |
| `success` | `#4f9b65` | 已发布状态 |
| `warning` | `#b9832f` | 待发布、未保存变更 |
| `danger` | `#c64545` | 删除、校验错误、归档 |
| `focus-ring` | `rgba(204,120,92,0.28)` | 键盘焦点环 |

不使用深森林绿 `#2C4A3A` 和金色 `#967B3E` 作为品牌色。不给每个 section 或卡片画边框。

### 4.2 字体

| 角色 | 字体 | 用途 |
|---|---|---|
| Display（标题） | `Tiempos Headline` / `Cormorant Garamond` / `Noto Serif SC` | 文档 h1、section h2 |
| Body / UI | `Inter` / `Noto Sans SC` / system-ui | 正文、按钮、输入框、导航、后台 |
| Mono | `JetBrains Mono` / `SF Mono` | 代码、slug、命令、diff |

**字号阶梯**：

| Token | 大小 | 字重 | 行高 | 用途 |
|---|---:|---:|---:|---|
| `display-lg` | 44px | 400 | 1.12 | 公开文档主标题（h1） |
| `display-md` | 32px | 400 | 1.20 | section 标题（h2） |
| `display-sm` | 24px | 400 | 1.28 | callout 标题、小节标题（h3） |
| `title-lg` | 22px | 500 | 1.35 | 后台页面标题 |
| `title-md` | 18px | 500 | 1.45 | 列表项标题、表单组标题 |
| `body-reading` | 17px | 400 | 1.76 | 公开阅读页正文 |
| `body` | 16px | 400 | 1.68 | 后台正文、表单说明 |
| `body-sm` | 14px | 400 | 1.55 | sidebar、footer、helper |
| `meta` | 13px | 400 | 1.45 | 日期、作者、版本 |
| `label` | 12px | 500 | 1.35 | badge、tab、状态 |
| `button` | 14px | 500 | 1.20 | 按钮 |
| `mono` | 13px | 400 | 1.65 | 代码和命令 |

- **正文字重 400**，不使用 300。
- **letter-spacing 始终为 0**，不靠负字距制造风格。
- **标题用衬线字体，UI 和正文用无衬线**。

### 4.3 布局

- 公开阅读页内容最大宽度 **760px**，居中。
- 左右 padding：桌面 **48px**，平板 **32px**，移动端 **20px**。
- 页面大 section 上下默认 **96px**，紧凑 section **72px**。
- 文档内 h2 距离上一段内容 72px，标题后到正文 20–24px，段落之间 16–20px。
- callout 和内容面板内边距 **24px**，大型后台面板 **32px**。
- **不使用 section 分隔线**，层次来自留白和字号。

### 4.4 组件规范

**callout（info-callout）**：`surface-soft` 底色，无左竖线，无边框，8px 圆角，padding 20px 24px。

**内容卡片（content-card）**：`surface-soft` 底色，无边框，8px 圆角，padding 24px。仅用于必须被成组识别的信息。

**代码块（code-block）**：`surface-dark` 暗色底，`ink-on-dark` 文字，13px mono，padding 20–24px，8px 圆角。复制按钮 ghost icon button，hover 时出现。

**表格**：不画完整外框，只画行分隔（`hairline-soft`）。表头使用 `label` 字号和 muted 文字色。

**按钮**：
- 主按钮：`primary` 底白字，40px 高，8px 圆角，`button` 字号，hover `primary-hover`，pressed `primary-pressed` + scale(0.97)
- 次按钮：`surface-soft` 底，hover 加深到 `surface-card`，无硬边框
- ghost 按钮：透明底，hover 出现极淡珊瑚底（`link-hover-fill`）
- 危险按钮：`danger-surface` 底 `danger` 字，不使用红底白字

**输入框**：`canvas` 底色，1px `hairline` 边框，8px 圆角，40px 高，focus 时 `primary` 边框 + 3px `focus-ring`。

**引用块**：无金色竖线。使用 `surface-soft` 或行内斜体处理，风格由系统 CSS 决定。

### 4.5 交互动效

| 元素 | 行为 |
|---|---|
| 顶部进度条 | **不使用** |
| 按钮按压 | `transform: scale(0.97)`，120ms |
| 大型 section 入场 | fade-up：opacity 0→1，translateY 12px→0，`cubic-bezier(0.22, 1, 0.36, 1)`，后台列表不做入场动画 |
| 链接 hover | `link-hover-fill` 极淡珊瑚底，180ms |
| 模态/抽屉 | 从右侧或顶部轻入，320ms ease-out，遮罩 `surface-overlay` |
| 编辑态 | 可编辑区 1px 虚线珊瑚色（primary 46% alpha），hover 加 5% 珊瑚底，非编辑模式不出现 |
| 保存反馈 | 右下角 toast，暗色底浅色字，2.5s 淡出 |
| 顶栏粘性 | backdrop-filter blur(18px)，半透明 canvas 底 |
| 锚点跳转 | 平滑滚动，scroll offset = nav 高度 + 24px |
| 路由切换 | 420ms 内，内容轻微 fade + translateY 8px |
| reduced-motion | 所有动画降到 0.01ms，transform 置 none |

整体基调：**暖白、克制、编辑感**。拒绝渐变、拒绝装饰光斑、拒绝高饱和色。不用 emoji 当 UI 控件。

## 五、非功能需求

### 5.1 性能

- 文档阅读页首屏加载 < 1.5s（在 4G 网络下，文档 ≤ 10KB）。
- 编辑页交互响应（点击/输入/保存）< 200ms。
- 列表与目录树即使 500 篇文档也应顺滑。

### 5.2 响应式

- 桌面端（≥ 1024px）：完整体验。
- 平板端（768–1024px）：阅读页正常，管理后台侧边栏可折叠。
- 移动端（< 768px）：仅阅读，编辑入口禁用并提示"请用桌面端编辑"。

### 5.3 可访问性

- 关键交互可键盘操作（Tab 顺序合理）。
- 焦点态明显（珊瑚色 focus ring，`rgba(204,120,92,0.28)`，3px）。
- 颜色对比度满足 WCAG AA。
- 表单元素有 label，图标按钮有 `aria-label`。

### 5.4 浏览器支持

- Chrome / Edge / Safari / Firefox 最近两个大版本。
- 不支持 IE。
- 移动端 Safari iOS 15+、Chrome Android 最近两个版本。

### 5.5 数据与安全

- 所有提交内容服务端做 HTML 清洗（bleach），白名单标签与属性。
- 密码用 Django 默认 PBKDF2 / Argon2 哈希存储。
- HTTPS 强制（Caddy 自动签证）。
- SQLite 文件**每日自动备份**到非 web 目录，保留 30 天。
- 管理后台所有写操作记 audit log（谁、何时、改了什么）。
