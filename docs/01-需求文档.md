# 内部文档系统 · 需求文档

> 一个具有编辑感的、可托管 AI 生成内容的、可控的小型文档系统。
> 不是 Notion，不是 WordPress，不是 GitBook。

---

## 一、项目定位

### 1.1 一句话定义

一个供个人与 4–5 人小团队使用的内部文档系统，承载由 AI（Codex / Claude Code）生成的、视觉风格统一的 HTML 文档页面，支持在线文字编辑、版本管理与按需对外发布。

### 1.2 它要解决什么

- AI 生成的文档页面各自为政、风格漂移 → **由系统强制套用统一框架与设计 token**。
- 文档内容更新需要改 HTML 源码 → **页面上点击即编辑、改完即存，不碰源码**。
- 团队协作无法回溯改动 → **每次保存都是一个版本，可对比、可回退**。
- 内部/对外混在一起 → **草稿默认私密，主动发布后才公开可访问**。

### 1.3 不做的事

- 不做多人实时协同（4–5 人异步编辑足够）。
- 不做 Notion 式自由块编辑（结构由 HTML 决定，只改文字）。
- 不做评论、@、订阅、通知（保持轻）。
- 不做企业级 SSO、审批流、组织架构。
- 不做移动端编辑（移动端只读）。

## 二、角色与权限

### 2.1 三类角色

**管理员（Admin）**
- 唯一最高权限账号（项目所有者）。
- 拥有全部内容的增/删/改/发布/回退/账号管理权限。
- 可指派其他用户为编辑者。

**编辑者（Editor）**
- 团队成员账号（4–5 人）。
- 可登录查看全部内容（含草稿），可编辑分配给自己的文档。
- 不可删除、不可发布、不可管理账号。

**访客（Public）**
- 无需登录。
- 仅能访问已发布的公开链接。
- 看不到草稿、看不到编辑入口、看不到版本历史。

### 2.2 访问入口

| 路径 | 谁能看 | 说明 |
|---|---|---|
| `/d/<slug>` | 所有人（已发布的） | 公开文档阅读页 |
| `/login` | 所有人 | 不在主导航暴露，需知道地址 |
| `/admin/` | 管理员 / 编辑者 | 登录后的管理后台 |
| `/admin/edit/<id>` | 管理员 / 编辑者 | 编辑界面 |

公开页面**完全不出现"登录"链接**——保持对外的纯净感。管理者通过书签或自己记住 `/login` 进入。

## 三、核心功能

### 3.1 文档浏览（公开端）

每篇已发布文档对应一个独立 URL（`/d/<slug>`）。打开后是一个纯粹的阅读体验：

- 顶部 2px 金色进度条随滚动推进。
- 主标题用衬线字体，居中或左对齐由文档自身决定。
- 内容主区域 840px 宽度，左右各 48px padding。
- 段落、引用、卡片、对比块等元素严格遵循设计 token。
- 滚动时元素轻量 fade-up 进入，不夸张。
- 页脚极简：仅显示最后更新日期，无 logo、无导航。

阅读页**没有任何登录入口、编辑入口**。读者就是读者。

### 3.2 目录与导航（管理端）

登录后进入 `/admin/`，左侧是固定侧边栏，右侧是当前选中文档的预览/编辑区。

**侧边栏**：
- 顶部：当前用户、搜索框（⌘K 唤起）。
- 中部：文档目录树，支持多级嵌套、拖拽排序。
- 文档项右侧 hover 显示三个动作图标（编辑 / 复制链接 / 更多）。
- 每篇文档前用小圆点标识状态：实心绿 = 已发布、空心 = 草稿、红 = 已归档。
- 底部：新建按钮（"+ 新文档"）+ 设置入口。

**目录树交互**：
- 点击文件夹展开/收起，状态在 session 内保持。
- 拖拽改变顺序与父子关系，松手立即保存（无"保存目录"按钮）。
- 右键（或长按）弹出菜单：重命名、新增子节点、移动、删除。

### 3.3 编辑体验（核心）

**进入编辑模式**：
- 文档预览页右上角"编辑"按钮，点击进入编辑模式。
- 进入后：所有 `data-editable` 区域出现 1px 虚线金色边框，左上角小标签写"可编辑"。
- 不可编辑区域（布局、装饰图、装饰编号）保持原样，鼠标悬停时显示锁标。
- 工具栏从顶部缓缓滑下，不遮挡内容。

**编辑操作**：
- 点击任意可编辑区域，光标即就位，可直接打字。
- 工具栏极简：**加粗、斜体、链接、清除格式、撤销、重做、保存**。
- 不提供改字体、改颜色、改大小的功能——风格由系统决定，不由编辑者决定。
- 按 `⌘S` / `Ctrl+S` 保存。
- 离开编辑模式前如有未保存改动，弹出确认提示。

**自动保存**：
- 停止打字 3 秒后自动保存为"自动版本"（不计入正式历史）。
- 手动保存创建"命名版本"，进入版本历史。
- 自动版本仅保留最近 5 个，超出自动清理。

**保存反馈**：
- 右下角浮出 toast：米底深绿字，"已保存 · 14:32"。
- 2.5 秒后自动淡出。
- 失败时变为浅红底深红字，附"重试"按钮。

### 3.4 版本管理

每个文档维护一条完整的版本时间线：

**版本列表**：
- 编辑页右侧抽屉，点击"版本"图标滑出。
- 时间线倒序展示：每条版本包含时间、作者、变更摘要（增 X 字 / 删 Y 字）、可选的版本说明。
- 命名版本用衬线字体标注，自动版本用次级文字色淡化。

**版本对比**：
- 点击任一版本，主区域切换为"对比视图"。
- 左右双栏（移动端上下叠加）：左为当前版本，右为所选历史版本。
- 增加的文字用浅绿底色，删除的文字用浅红底色加删除线。
- 顶部一行操作：「← 返回 / 还原此版本 / 在新页面打开」。

**还原**：
- 点击"还原此版本"后，弹出二次确认（防误触）。
- 还原不是删除新版本，而是基于历史版本创建一个新版本，原有历史完整保留。

### 3.5 发布流程

**草稿与已发布**：
- 每篇文档有 `status` 字段：`draft` / `published` / `archived`。
- 默认 `draft`。
- 草稿仅登录后可见，公开 URL 返回 404。

**发布操作**：
- 编辑页顶部右侧的"发布"按钮。
- 点击后弹出抽屉确认：「即将发布到 `/d/<slug>`，所有人都将可访问」。
- 同步显示：当前 slug（可改）、首次发布的预计时间、是否生成新版本号。
- 确认后，状态切为 `published`，并自动创建一条命名版本（"发布于 YYYY-MM-DD"）。

**取消发布**：
- 已发布文档可"撤回"，状态回到 `draft`。
- 取消发布不删除文档，公开 URL 立即返回 404。

**链接管理**：
- slug 默认由文档标题生成（中文 → 拼音首字母 + 短随机后缀），可手动改。
- 改 slug 后旧链接自动 301 重定向到新链接（保留旧 slug 的映射记录）。

### 3.6 AI 生成内容的接入

这是整个系统**最特殊**的能力，区别于一切现成 CMS：

**接入方式**：
1. 用户在 Codex / Claude Code 中按预设模板生成 HTML 片段。
2. 在管理后台「+ 新文档」 → 选「从 HTML 导入」。
3. 粘贴 HTML 到大文本框，点"解析"。
4. 系统自动：
   - 清洗 HTML（剥离 `<script>`、危险属性，过 bleach）。
   - 校验是否符合系统 schema（必需 class、必需 `data-editable` 标记）。
   - 应用统一外层布局（840px 容器、字体、配色 token）。
   - 提取所有 `data-editable` 区域，存入版本表。
5. 校验通过后进入预览界面，可直接编辑、保存、发布。

**模板规范**（提供给 AI 的 prompt 模板，作为附件存在 `/templates/`）：
- 必须用系统提供的 class（`doc-section`, `doc-quote`, `doc-compare-pair` 等）。
- 文字内容必须包在 `data-editable="true"` 的元素里，且要有 `data-editable-id` 唯一标识。
- 不允许内联 `style` 属性。
- 不允许引入外部字体、外部 CSS。
- 不允许 `<script>`、`<iframe>`、`onclick` 等。

校验不通过时给出明确错误：哪一行、缺什么、怎么改。

### 3.7 搜索

- 全局搜索快捷键：`⌘K` / `Ctrl+K`。
- 唤起一个浮层（不是新页面），顶部居中，居中宽度 560px。
- 实时搜索（输入即查），结果分组：「文档标题」、「正文匹配」。
- 命中的关键词高亮。
- 上下箭头切换、回车跳转。
- 公开端不暴露搜索入口（保持阅读纯净）。

### 3.8 账号管理

**登录页**（`/login`）：
- 极简：居中卡片，账号 + 密码 + 登录按钮。
- 无注册入口（系统不开放注册）。
- 无"忘记密码"链接（管理员后台重置）。

**账号设置**（管理员）：
- `/admin/users` 路径下管理团队成员账号。
- 可新建、禁用、重置密码、删除。
- 每个账号可指定角色（admin / editor）。

**会话**：
- 登录默认 30 天 session（可勾选"本机记住"）。
- 修改密码后所有其他设备登录失效。

## 四、界面规范

### 4.1 配色 Token

| 用途 | 色值 | 备注 |
|---|---|---|
| 主背景 | `#F5F3EE` | 暖米色 |
| 表面 / 卡片 | `#FFFFFF` | |
| 主文字 | `#1A1814` | |
| 次级文字 | `#6D6A64` | |
| 辅助文字 | `#A09C95` | |
| 强调色 | `#2C4A3A` | 深森林绿，主行动按钮 |
| 金色点缀 | `#967B3E` | 进度条、引用线、装饰编号 |
| 边框 | `#E2DED6` | 卡片、输入框 |
| 浅边框 | `#EDEAE4` | section 分隔线 |
| 错误背景 | `#FBF3F3` | |
| 错误文字 | `#7A2E2E` | |
| 成功文字 | `#3F5F3A` | 补充 |
| 警告文字 | `#8A6A2E` | 补充 |
| 链接 hover 底 | `rgba(44,74,58,0.08)` | 补充 |
| 焦点 ring | `rgba(150,123,62,0.35)` | 补充 |

### 4.2 字体

- **标题**：`DM Serif Display`（英文/数字斜体装饰）+ `Noto Serif SC`（中文衬线）
- **正文**：`Noto Sans SC`，字重 `300`，行高 `1.8`
- **装饰编号**：`DM Serif Display italic`，浅色 `#E2DED6` 作 hero numerals

**字号阶梯**（基准 16px）：

| 角色 | 大小 | 说明 |
|---|---|---|
| H1 | 2.5rem (40px) | 文档主标题 |
| H2 | 1.875rem (30px) | section 标题 |
| H3 | 1.375rem (22px) | 子节标题 |
| 正文 | 1rem (16px) | |
| 辅助 | 0.875rem (14px) | meta 信息 |
| 标签 | 0.6875rem (11px) | 全大写，letter-spacing `0.08em` |

### 4.3 布局

- 内容最大宽度 **840px**，居中。
- 左右 padding **48px**（移动端 20px）。
- Section 上下 padding **56px**。
- Section 之间 1px `#E2DED6` 分隔线。

### 4.4 组件规范

**卡片**：白底，1px `#E2DED6` 边框，圆角 10px，padding 22px，**无阴影**。

**引用块**：左侧 3px 金色 (`#967B3E`) 竖线，白底，`Noto Serif SC`，字重 300。

**对比双列**：左白底、右浅红 `#FBF3F3` 并排，顶部 11px 全大写标签。

**步骤列表**：34px 深绿圆点编号，1px 竖线连接，右侧内容 padding-bottom 28px。

**标签 tag**：11px，白底，1px 边框，圆角 4px，padding `4px 9px`。

**按钮**：
- 主按钮：深绿底白字，圆角 6px，padding `10px 18px`，hover `#1F3A2D`
- 次按钮：白底 1px 边框，hover 底 `#FAF8F4`
- 文字按钮：仅金色文字 + 下划线，hover 加深

**输入框**：白底，1px `#E2DED6` 边框，圆角 6px，padding `10px 14px`，focus 金色 2px ring（半透明）。

### 4.5 交互动效

| 元素 | 行为 |
|---|---|
| 顶部进度条 | 2px 金色固定，滚动实时推进 |
| 元素入场 | fade-up：opacity 0→1，translateY 18px→0，`cubic-bezier(0.22, 1, 0.36, 1)`，0.55s |
| 链接 hover | 底色淡淡透明绿铺出，180ms |
| 模态/抽屉 | 从右滑入 320ms，遮罩 `rgba(26,24,20,0.4)` |
| 编辑态 | 可编辑区 1px 虚线金边 + 左上"可编辑"小标签 |
| 保存反馈 | 右下角 toast，米底深绿字，2.5s 淡出 |

整体基调：**温暖、克制、编辑感**。拒绝渐变、拒绝多层阴影、拒绝高饱和色。

## 五、非功能需求

### 5.1 性能

- 文档阅读页首屏加载 < 1.5s（在 4G 网络下，文档 ≤ 10KB）。
- 编辑页交互响应（点击/输入/保存）< 200ms。
- 列表与目录树即使 500 篇文档也应顺滑。

### 5.2 响应式

- 桌面端（≥ 1024px）：完整体验。
- 平板端（768–1024px）：阅读页正常，管理后台侧边栏可折叠。
- 移动端（< 768px）：仅阅读，编辑入口禁用并提示"请用桌面端编辑"。

### 5.3 可访问性

- 关键交互可键盘操作（Tab 顺序合理）。
- 焦点态明显（金色 ring）。
- 颜色对比度满足 WCAG AA。
- 表单元素有 label，图标按钮有 `aria-label`。

### 5.4 浏览器支持

- Chrome / Edge / Safari / Firefox 最近两个大版本。
- 不支持 IE。
- 移动端 Safari iOS 15+、Chrome Android 最近两个版本。

### 5.5 数据与安全

- 所有提交内容服务端做 HTML 清洗（bleach），白名单标签与属性。
- 密码用 Django 默认 PBKDF2 / Argon2 哈希存储。
- HTTPS 强制（Caddy 自动签证）。
- SQLite 文件**每日自动备份**到非 web 目录，保留 30 天。
- 管理后台所有写操作记 audit log（谁、何时、改了什么）。
