为什么不是 Markdown

HTML 优先

Markdown 适合写，不适合读。它把所有文档压成同一种"等宽语气"， 层级靠井号，结构靠空行 — 一旦内容超出"标题 + 段落 + 列表"， MD 就开始用更多语法补丁去模拟它本不擅长的事情。HTML 不需要模拟。

Markdown 的天花板

  • 版式被渲染器决定，作者无法控制留白与节奏
  • 表格只能"画框格"，无法做对比、卡片、状态徽章
  • callout/警告/步骤条要靠各家方言扩展
  • 图与文不能并排，长正文只剩竖向阅读
  • 代码高亮之外，几乎没有视觉表达

HTML 给的余量

  • 语义标签清晰：section / aside / figure / details
  • 排版交给设计系统：字体、节奏、留白可控
  • 组件化展示：callout、对比、步骤、徽章、表格、代码窗
  • 响应式与可访问性原生支持
  • 同一份内容能在文档站、邮件、PDF 里复用

设计取舍：编辑器允许任意结构的 HTML，由白名单清洗后存档 — 不限定模板，但保证安全与一致的视觉语言。
